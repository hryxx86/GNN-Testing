# 前言 + 背景

## 什么是股票预测？我们到底想做什么？

想象你每天早上打开手机，看到一条新闻："苹果公司发布新iPhone，销量超预期"。你的直觉告诉你：这是好消息，苹果的股票明天可能会涨。但问题是——你的直觉准吗？如果我们能让计算机自动读懂几百万条这样的新闻，然后预测股票明天是涨还是跌，那岂不是就能赚钱了？

这就是我们这个项目的核心目标：**用人工智能（AI）自动阅读新闻，预测股票的涨跌方向**。

具体来说，我们的任务定义是：给定一条关于某只股票的新闻，预测该股票**下一个交易日的收盘价**相比今天是上涨（标签 = 1）还是下跌（标签 = 0）。这是一个**二分类问题**（Binary Classification，将事物分成两类的任务——涨或跌）。

## 什么是图神经网络（GNN）？为什么用"图"来分析股票？

**图**（Graph）在这里不是指图片，而是数学中的一种数据结构。一个图由**节点**（Node，代表一个事物）和**边**（Edge，代表两个事物之间的关系）组成。举个例子：把每只股票想象成一个圆圈（节点），如果两只股票的价格经常同涨同跌（相关性高），就在它们之间画一条线（边）。这样 502 只股票就形成了一张"关系网"。

为什么股票适合用图来分析？因为股票不是孤立的。苹果涨了，它的供应商台积电可能也会涨；能源板块集体下跌时，埃克森美孚和雪佛龙往往一起跌。这些**股票之间的联动关系**正是图能捕捉的信息。

**图神经网络**（Graph Neural Network，简称 GNN）是一种专门在图上学习的深度学习模型。它的核心思想是**消息传递**（Message Passing）：每个节点收集邻居的信息，综合之后更新自己的特征。就像"近朱者赤"——一只股票的预测不仅取决于它自己的新闻，还取决于与它关联的其他股票的信息。

## 什么是 S&P 500？为什么是 502 只股票？

**S&P 500**（Standard & Poor's 500，标准普尔 500 指数）是美国股市最重要的指数之一，包含了美国市值最大、交易最活跃的约 500 家上市公司，涵盖苹果、微软、亚马逊、摩根大通等。选择 S&P 500 是因为：（1）它是全球研究最多的股票市场，结论有普遍意义；（2）这些公司新闻数据丰富。至于为什么是 502 而不是 500——因为成分股会动态调整，我们取的是数据覆盖期内的全部有效成分股，恰好有 502 只。

## 我们有什么数据？

我们的数据来自三个来源：

1. **170 万条新闻事件**：来自 EODHD（一个金融数据供应商），涵盖 2021 年至 2025 年。每条新闻记录了标题、日期、涉及的股票代码。
2. **FinBERT 嵌入向量**（Embedding，将文本转换成计算机能理解的数字向量）：我们用一个叫 FinBERT 的预训练语言模型（Pre-trained Language Model，在海量金融文本上训练好的 AI 模型）处理每条新闻的标题，将其转化为一个 768 维的向量（768-dimensional vector，768 个数字组成的列表，每个数字捕捉标题的某个语义特征）。同时，FinBERT 还输出 3 个情感得分：正面概率、负面概率、中性概率。
3. **股票价格数据**：5 年的每日收盘价，用来计算收益率（Return，股价的变化百分比）、构建相关性图、生成标签。

## 小规模先导实验的结果

在全规模实验之前，我们先做了一个**先导实验**（Pilot Study，小范围的预实验，验证方法是否可行）：只选了 9 只最有代表性的股票（苹果、微软等），用 480 条来自 Factiva（另一个新闻数据库）的高质量新闻，用 MiniLM（另一种语言模型）生成 384 维的嵌入向量。

结果令人振奋：
- **Logistic Regression**（逻辑回归，一种最简单的分类模型）的 AUC（Area Under Curve，曲线下面积，衡量模型好坏的指标，0.5 = 随机瞎猜，1.0 = 完美预测）达到 **0.62**
- **GraphSAGE**（一种 GNN 模型）的 AUC 达到 **0.64**

这意味着模型确实比瞎猜好了不少，图结构（GNN）还额外提升了 2 个百分点。

## 为什么要放大规模？风险在哪里？

先导实验虽然结果不错，但存在严重的局限性：9 只股票、480 条新闻、单一来源。这么小的样本量可能产生**统计假象**（Statistical Artifact，由于数据太少导致的虚假结论）。如果想发表论文并得到可靠结论，必须在完整的 S&P 500（502 只股票、170 万条新闻）上验证。

但放大规模的风险也很明显：
- 502 只股票中大部分的新闻质量可能远不如 9 只顶级科技股
- 170 万条来自 EODHD 的新闻标题可能比 Factiva 的全文新闻信息量更少
- S&P 500 是全球交易量最大、**有效市场假说**（Efficient Market Hypothesis / EMH，指价格已经反映了所有公开信息，因此很难通过分析公开信息赚钱）最强的市场——新闻中的信息可能早就被大量机构投资者消化了

换句话说：先导实验的 AUC 0.64 有可能只是小数据上的幸运，全规模实验很可能看到 AUC 骤降。

## Cell 7: 构建异构图（HeteroData）—— 核心架构

这是整个实验最关键的一步。我们要把 170 万条新闻和 502 只股票组织成一张**异构图**（Heterogeneous Graph，包含不同类型节点和不同类型边的图）。

### 什么是异构图？

普通的图（同构图）只有一种节点、一种边。但在我们的场景中，**新闻**和**股票**是完全不同的东西，它们之间的关系也有多种。所以我们构建一个异构图，包含：

**两种节点**：
1. **新闻节点**（News Node）：每条新闻是一个节点，特征维度 = 780（768 维 FinBERT 嵌入 + 3 维情感得分 + 9 维动量特征，后面解释）
2. **股票节点**（Stock Node）：每只股票是一个节点，特征维度 = 12（11 维行业 one-hot 编码 + 1 维对数市值）

**三种边**：
1. **新闻→股票**（relates_to）：如果一条新闻提到了某只股票，就在它们之间连一条边。170 万条新闻 = 170 万条这种边。
2. **股票↔股票（相关性）**（correlated_with）：如果两只股票的历史价格高度相关（Pearson 相关系数绝对值 > 0.6），就连一条边。
3. **股票↔股票（同行业）**（same_sector）：如果两只股票属于同一个 GICS 行业（如都是科技股），就连一条边。

### 新闻节点特征的构建

```python
# phase_c_model_training.ipynb, Cell 7
# News node features: FinBERT(768) + sentiment(3) + momentum(9) = 780-dim
news_x = np.concatenate([emb[valid_idx], sent[valid_idx], momentum_features[valid_idx]], axis=1)
data['news'].x = torch.from_numpy(news_x.astype(np.float32))
```

每条新闻的特征是三部分拼接而成：
- **FinBERT 嵌入向量（768 维）**：标题的语义信息，比如"苹果发布新品"和"苹果股票回购"会被映射到向量空间中不同但相近的位置
- **情感得分（3 维）**：正面/负面/中性概率
- **动量特征（9 维）**：该股票在新闻发布前的价格走势，包括 3 个时间窗口（5天/10天/21天）各自的均值收益率、波动率（Volatility，价格波动的剧烈程度）和动量（Momentum，一段时间内的累计涨跌幅）

`np.concatenate(..., axis=1)` 的意思是将这三个矩阵沿列方向拼接——768 + 3 + 9 = 780 维。

### 股票节点特征的构建

```python
# phase_c_model_training.ipynb, Cell 7
# Stock node features: GICS one-hot(11) + log_market_cap(1) = 12-dim
sector_to_idx = {s: i for i, s in enumerate(unique_sectors)}
stock_feat = []
for tk in valid_tickers:
    oh = [0.0] * len(unique_sectors)
    s = sector_map.get(tk, 'Unknown')
    if s in sector_to_idx:
        oh[sector_to_idx[s]] = 1.0
    cap = mcaps.get(tk, 0)
    log_cap = np.log1p(max(float(cap), 0)) / 30.0
    stock_feat.append(oh + [log_cap])
data['stock'].x = torch.tensor(stock_feat, dtype=torch.float32)
```

每只股票的特征由两部分组成：
- **GICS 行业 one-hot 编码（11 维）**：GICS（Global Industry Classification Standard，全球行业分类标准）将所有公司分为 11 个大行业（科技、金融、医疗、能源等）。One-hot 编码（独热编码）是一种表示类别的方式——如果一只股票属于科技行业，就在"科技"那个位置填 1，其余 10 个位置填 0。
- **对数市值（1 维）**：`np.log1p(cap) / 30.0`，先取对数（因为市值跨度极大，苹果 3 万亿美元 vs 某小公司 100 亿美元，对数能压缩这个差距），再除以 30 做归一化（Normalization，将数值缩放到一个合理的范围，避免某个特征因为数值太大而主导模型）。

### 相关性边的构建

```python
# phase_c_model_training.ipynb, Cell 7
prices_train = prices.loc[prices.index <= cutoff, valid_tickers]
rets = prices_train.pct_change(fill_method=None).iloc[1:]
corr_mat = rets.corr().values
thr = PARAMS['corr_threshold']  # 0.6
mask_corr = np.abs(corr_mat) > thr
np.fill_diagonal(mask_corr, False)
c_src, c_dst = np.where(mask_corr)
```

这段代码计算股票之间的 **Pearson 相关系数**（衡量两个变量线性相关程度的指标，范围 -1 到 +1，+1 = 完全同方向变动，-1 = 完全反方向变动，0 = 无关）。具体步骤：
1. 取 2024 年底之前的价格数据（避免数据泄漏）
2. `pct_change()` 计算每日收益率（今天收盘价相比昨天涨跌了百分之几）
3. `rets.corr()` 计算所有股票两两之间的相关系数矩阵（502 × 502）
4. 只保留绝对值 > 0.6 的配对（阈值 0.6 意味着只有强相关的股票才连边）
5. `np.fill_diagonal(mask_corr, False)` 把对角线去掉——一只股票与自己的相关系数当然是 1，但我们不需要自己连自己

### 时间划分与双向边

```python
# phase_c_model_training.ipynb, Cell 7
data['news'].train_mask = torch.tensor((dates <= train_end).values)
data['news'].val_mask   = torch.tensor(((dates > train_end) & (dates <= val_end)).values)
data['news'].test_mask  = torch.tensor((dates > val_end).values)

data = T.ToUndirected()(data)
```

时间划分采用**严格的时间顺序**（Temporal Split）：训练集 → 验证集 → 测试集，绝不打乱。这是金融预测的铁律——不能用未来数据训练去预测过去。

`T.ToUndirected()` 将所有单向边变成双向边。为什么？因为在 GNN 的消息传递机制中，如果只有"新闻→股票"的边，那新闻节点就收不到来自股票节点的信息。我们希望信息能双向流动：新闻影响股票的预测，同时一只股票的行业和关联信息也能反过来帮助理解与之相关的新闻。

## Cell 9: 基准模型（Baselines B1-B5）

在测试复杂的 GNN 之前，我们先要建立**基准线**（Baseline，最简单的参照模型，用来衡量复杂模型到底好了多少）。

### B1: 逻辑回归 + FinBERT 768 维

**逻辑回归**（Logistic Regression，简称 LR）是最简单的分类模型之一。它本质上就是一个线性方程：把 768 个数字各乘以一个权重，加起来，通过一个 S 形函数（Sigmoid）压缩到 0-1 之间，输出"涨的概率"。如果概率 > 0.5 就预测涨，否则预测跌。

这个基准的意义是：FinBERT 的 768 维嵌入向量本身包含多少预测信号？如果连最简单的线性模型都无法提取信号，那更复杂的 GNN 也不太可能成功。

### B2: 逻辑回归 + 情感 3 维

只用 3 个情感得分（正面、负面、中性概率）。这个基准测试的是：抛开复杂的 768 维语义信息，仅凭最直觉的"正面/负面/中性"分类，能预测涨跌吗？

### B3: 逻辑回归 + 情感 + 动量（12 维）

3 维情感 + 9 维动量特征。测试的是：如果把新闻情感与价格趋势信息结合，会不会好一些？

### B4: 逻辑回归 + 动量（9 维）

只用价格动量，完全不看新闻。这是一个极其重要的对照：如果 B4 比 B1 好，就说明**价格信号比新闻信号更有用**；如果 B4 和 B1 一样差，说明两种信号都没用。

### B5: XGBoost（情感 + 动量）

**XGBoost**（eXtreme Gradient Boosting，极端梯度提升）是一种集成学习模型（Ensemble Model，组合多个简单模型来做出更好预测）。它的核心思想是训练一系列"决策树"（Decision Tree，通过一连串"如果...则..."规则做决定的模型），每棵树都在前一棵树犯错的地方重点改进。XGBoost 在各种表格数据竞赛（如 Kaggle）中常常击败深度学习模型，是业界公认的强力基准。如果 XGBoost 都预测不了，说明信号确实不存在。

## Cell 10-11: GNN 模型与消融实验

### GraphSAGE 是什么？

```python
# phase_c_model_training.ipynb, Cell 10
class GNN(torch.nn.Module):
    def __init__(self, hidden_channels, dropout=0.3):
        super().__init__()
        self.conv1 = SAGEConv((-1, -1), hidden_channels)
        self.conv2 = SAGEConv((-1, -1), hidden_channels)
        self.lin = torch.nn.Linear(hidden_channels, 1)
        self.dropout = dropout

    def forward(self, x, edge_index):
        x = self.conv1(x, edge_index).relu()
        x = F.dropout(x, p=self.dropout, training=self.training)
        x = self.conv2(x, edge_index).relu()
        return self.lin(x)
```

**GraphSAGE**（Graph SAmple and aggreGatE，图采样与聚合）是一种经典的 GNN 架构。它的工作方式是：

1. **第一层（conv1）**：每个节点收集自己一跳邻居的特征，聚合后更新自己的表示。比如一条新闻节点收集它所提及的股票节点的信息；一只股票节点收集所有提及它的新闻节点的信息。
2. `.relu()`：ReLU 激活函数（Rectified Linear Unit），简单来说就是"负数变零，正数不变"，给模型引入非线性（Non-linearity，让模型能学习复杂的、非直线的关系）。
3. `F.dropout`：训练时随机关掉 30% 的神经元，防止过拟合。
4. **第二层（conv2）**：重复一次。经过两层之后，每个节点已经"看到"了两跳之内的邻居信息。
5. **线性层（lin）**：将 64 维的隐藏表示映射为 1 个输出值——即预测的涨跌分数。

### to_hetero() 是什么？

```python
model = to_hetero(base_model, graph_data.metadata(), aggr='mean')
```

`to_hetero()` 是 PyTorch Geometric（PyG，一个专门处理图数据的 PyTorch 扩展库）提供的工具。因为我们有两种节点（新闻、股票）和三种边（提及、相关性、行业），原始的 GraphSAGE 是为同构图设计的，`to_hetero()` 自动将其扩展为可以处理异构图的版本——为每种边类型创建独立的参数。

### 损失函数与早停

```python
criterion = torch.nn.BCEWithLogitsLoss()
```

**BCEWithLogitsLoss**（Binary Cross-Entropy with Logits Loss，带 logits 的二元交叉熵损失）：模型输出一个分数（logit），损失函数衡量这个分数与真实标签（0 或 1）之间的差距。训练过程就是不断调整模型参数，使这个损失越来越小。

**早停**（Early Stopping）是防止过拟合的重要技巧。具体做法：每个 epoch（训练一遍所有数据）后，在验证集上检查 AUC。如果连续 15 个 epoch（patience=15）验证集 AUC 都没有改善，就停止训练，使用之前最好的模型。这就像考试前复习——到了一定程度再继续死磕反而分数下降，不如适时收手。

### 四个消融实验（A1-A3, Full）

**消融实验**（Ablation Study，逐一去掉系统的某个部分来观察其影响，类比"拆零件看缺了哪个车就不能动"）：

- **A1: 只保留新闻→股票边**——GNN 只能看到新闻与股票的直接关联
- **A2: 新闻→股票 + 相关性边**——加入股票之间的价格相关性
- **A3: 新闻→股票 + 行业边**——加入同行业关系
- **Full: 所有三种边**——完整模型

### 全批次训练

我们采用**全批次训练**（Full-Batch Training），即每个 epoch 都将整张图（全部 170 万新闻节点 + 502 股票节点 + 所有边）一次性加载到 A100 GPU 的 80GB 显存中。这样做的好处是：（1）每次更新模型参数时看到的是完整数据，不存在采样噪声；（2）避免了需要安装复杂的 torch-sparse 库。代价是需要巨大的 GPU 显存，但 A100 80GB 完全够用。

## 结果：所有 AUC 约等于 0.50

| 实验 | 验证集 AUC | 测试集 AUC |
|------|-----------|-----------|
| B1: LR + FinBERT (768维) | 0.5018 | 0.4976 |
| B2: LR + 情感 (3维) | 0.5044 | 0.5027 |
| A1: GNN 仅新闻→股票 | 0.5085 | 0.4913 |
| A2: + 相关性边 | 0.5122 | 0.4949 |
| A3: + 行业边 | 0.5133 | 0.4961 |
| Full: 所有边 | 0.5133 | 0.5069 |

**所有实验的测试集 AUC 都在 0.49-0.51 之间。**

AUC 0.50 意味着什么？它意味着模型的预测能力和**抛硬币完全一样**。如果你闭着眼睛随机猜涨跌，平均准确率就是 50%，AUC 就是 0.50。我们投入了 170 万条新闻、用了最先进的 FinBERT 语言模型、构建了精心设计的异构图神经网络，最终得到的结论是：**和瞎猜没有区别**。

这和先导实验的 AUC 0.62-0.64 形成了鲜明对比。从情感上说，这是一个沉重的打击——几周的工作都没有产生任何有意义的信号。但从科学角度来说，这个 negative result（否定性结果）本身就是重要的发现：它揭示了先导实验的 0.64 很可能确实是小样本的统计幻觉。

# 诊断 — 为什么全部失败？

在看到所有 AUC 约 0.50 的结果后，我们没有盲目地换模型或调参数，而是首先做了**系统性诊断**（Systematic Diagnostics）——从数据和模型两个层面找出失败的根本原因。

## Cell 17: D.1 数据层面诊断

### 分析 1：标签噪声——26.5% 的标签本质是噪声

```python
# phase_c_model_training.ipynb, Cell 17
abs_ret = np.abs(returns_raw)
tiny_pct = (abs_ret < 0.005).mean()   # |ret| < 0.5%
small_pct = (abs_ret < 0.01).mean()   # |ret| < 1%
```

我们的标签定义是：如果明天收盘价比今天涨了就是 1（涨），跌了就是 0（跌）。但问题是：如果一只股票明天只涨了 0.01%（即 100 元变成 100.01 元），这算"涨"吗？从交易角度看，这个涨幅完全被交易手续费吃掉了，没有任何实际意义。

诊断结果：
- **26.5%** 的新闻事件对应的股票次日涨跌幅在 ±0.5% 以内——这些标签本质上是随机的"噪声"
- **48.0%** 在 ±1.0% 以内

也就是说，我们的标签中有将近**一半**是"掷硬币"级别的。任何模型在面对这么大比例的噪声标签时，都很难学到有用的模式。

各个收益率区间的正面标签比例：

| 收益率区间 | 事件数量 | 正面标签比例 |
|-----------|---------|------------|
| < 0.5% | 445K | 0.494（几乎完美对半）|
| 0.5-1% | 365K | 0.508 |
| 1-2% | 456K | 0.532 |
| 2-5% | 358K | 0.522 |
| > 5% | 69K | 0.525 |

即使在涨跌幅超过 5% 的"大事件"中，正面标签比例也只有 52.5%——仍然非常接近随机。

### 分析 2：情感与涨跌的对齐度——FinBERT 的情感判断几乎无法预测涨跌

这是最关键的诊断。直觉上，正面新闻应该对应股票上涨，负面新闻应该对应下跌。但实际数据告诉我们：

- **高置信度正面新闻**（FinBERT 判断"正面"的概率 > 0.7）：实际上涨的比例 = **51.6%**（仅比随机的 50% 高了 1.6 个百分点）
- **高置信度负面新闻**：实际下跌的比例 = **48.9%**（甚至略低于 50%，即负面新闻反而微微预示着上涨）

这意味着 FinBERT 对新闻标题的情感判断和股票涨跌之间几乎**没有任何关联**。即使 FinBERT 非常自信地说"这条新闻是正面的"，股票第二天涨的概率也只比 50% 高了可以忽略的一丁点。

### 分析 3：各行业统计——信息科技行业占据主导

| 行业 | 事件数量 | 正面比例 |
|------|---------|---------|
| Information Technology | 420K (24.8%) | ~50% |
| Consumer Discretionary | ... | ~50% |
| ... | ... | ~50% |

IT 行业贡献了将近 1/4 的新闻，但所有行业的正面标签比例都紧贴 50%。这说明问题不是某个特定行业拖了后腿——是**所有行业都一样**。

### 分析 4：时间稳定性——市场状态剧变

- **2022年 Q2**：正面比例 = 45.7%，平均收益 = -0.37%（熊市，Bear Market，指股市持续大幅下跌的阶段）
- **2023年 Q4**：正面比例 = 55.9%，平均收益 = +0.21%（牛市反弹，Rally）

市场存在明显的**政权更迭**（Regime Shift，市场从一种状态转换到另一种状态，如从牛市转为熊市）。在熊市中，好新闻也挡不住集体下跌；在牛市中，坏新闻也不影响集体上涨。我们的模型是在全部时间段上训练的静态模型，无法适应这种市场状态的剧烈变化。

## Cell 18: D.2 模型预测层面诊断

### 分析 5：模型无法区分涨跌——预测分数的分离度几乎为零

              test_events[test_events['true_label']==0]['score'].mean())
# 结果: separation = -0.00030

结果：两类事件的平均分数差距 = **-0.00030**，几乎为零。这意味着在模型眼中，明天会涨的新闻和明天会跌的新闻**长得一模一样**——它完全无法分辨。

### 分析 8：按收益率幅度切分——即使大事件也无法预测

## 最终结论

八项诊断分析得出一个统一的、无可辩驳的结论：

**FinBERT 的新闻标题级情感分析对 S&P 500 下一日收益率的预测能力为零。这不是模型的问题，不是数据处理的 bug，不是超参数没调好——是信号本身不存在。**

为什么？核心原因有三：

1. **有效市场假说（EMH）在起作用**：S&P 500 是全球最高效的市场。每条新闻发布的瞬间，成千上万的机构投资者、量化基金、算法交易系统已经在毫秒级别完成了交易。等我们第二天看收盘价的时候，新闻中的信息早已被完全消化。
2. **标题太短，信息量太少**：平均每条新闻标题只有约 15 个单词。FinBERT 的 768 维空间原本设计用来编码丰富的语义信息，但 15 个词根本喂不饱这 768 个维度——大部分维度都是噪声。
3. **情感 ≠ 涨跌方向**：新闻情感（正面/负面）和股票涨跌之间的关系远比直觉复杂。"苹果利润下滑"听起来是负面新闻，但如果市场预期更差，股票反而会涨——因为"没那么糟"就是好消息。这种**预期差**（Expectation Gap）让简单的情感分析完全失效。

# 文献调研 — 别人做得怎么样？

## 为什么要做文献调研

在 Phase C 全部实验 AUC ≈ 0.50 之后，我们面临一个关键问题：**这到底是代码写错了，还是这件事本身就做不到？**

为了回答这个问题，我们系统检索并分析了 6 篇近年发表的、同样尝试用 GNN + 自然语言处理（NLP）预测股票的论文。

## 6 篇论文横向对比

| 论文 | 发表场所 | 预测多少只股票 | 用了新闻吗？ | 最好成绩 |
|------|---------|-------------|------------|---------|
| THGNN | CIKM 2022 | 300-500 只（中国+美国市场） | **没有**，纯价格数据 | 收益排名（非方向预测） |
| DGRCL | ICAART 2025 | **1,026 只**（纳斯达克） | **没有**，价格+成交量 | 准确率 53.06% |
| DASF-Net | JRFM 2025 | **12 只**（4 个行业各选 3 只） | 用了 FinBERT | "MSE 降低 91.6%" |
| ChatGPT-GNN | KDD Workshop 2023 | **30 只**（道琼斯 30） | 用了 ChatGPT | F1 = 0.41 |
| Kengmegni 2024 | SSRN | 标普 500 | 用了 FinBERT | **结论：情感没有稳健预测力** |
| Sentiment-Size Nexus 2025 | JBA | 大/中/小盘指数 | 用了 Doc2Vec+SVM | 对指数有效，对个股无效 |

## 核心发现：股票数量才是最大的"作弊变量"

这 6 篇论文揭示了一个极其关键的规律——**预测的股票数量越少，论文报告的数字看起来越好看，但越不可信。**

**DASF-Net（12 只股票）**：只从标普 500 中挑选了 4 个行业、每个行业 3 只股票，总共 12 只。他们报告"MSE 降低 91.6%"。MSE（Mean Squared Error，均方误差）是衡量预测值和真实值之间差距的指标——降低 91.6% 听起来非常惊人。但这里有两个严重问题：第一，他们做的是价格回归（预测具体涨多少），不是方向预测（预测涨还是跌），这两件事的难度完全不同；第二，12 只股票可以被精心挑选。

**ChatGPT-GNN（30 只股票）**：用 ChatGPT 分析新闻标题，在道琼斯 30 指数的 30 只成分股上做三分类预测。报告加权 F1 = 0.41。F1 是一个综合衡量精确率和召回率的指标。三分类的随机基线是 0.33，所以 0.41 只比随机好了 8 个百分点——实际提升相当有限。

**DGRCL（1,026 只股票）**：这是唯一一篇和我们"量级"相当的论文。他们在 1,026 只纳斯达克股票上做次日方向预测，最终结果：**53.06% 的准确率**——也就是说，在 100 次预测中只比随机多猜对 3 次。我们的 AUC ≈ 0.50 换算成准确率大约是 50-51%，和他们处于**完全相同的水平**。

## 有效市场假说（EMH）

**有效市场假说**（Efficient Market Hypothesis，简称 EMH）是金融学中最重要的理论之一。它的核心观点是：**股价已经反映了所有可获得的信息**。如果一条新闻说苹果公司业绩超预期，在你打开新闻网站看到这条标题之前，市场上的高频交易算法（以毫秒级速度交易的计算机程序）和专业基金经理们已经买入了苹果股票，股价已经涨上去了。

标普 500 是全世界**效率最高**的市场：
- 每只成分股都被几十家投行的分析师覆盖
- 高频交易算法 24 小时监控新闻和数据
- 实证数据显示，**只有约 20% 的美国大盘股主动基金能跑赢指数**

**Kengmegni（2024）** 的研究明确证实：FinBERT 情感分数对标普 500 大盘股"缺乏稳健的预测力"。

## 结论：AUC ≈ 0.50 不是 Bug，是预期结果

综合 6 篇论文的证据，我们的 AUC ≈ 0.50 是以下 5 个因素共同作用的**必然结果**：

1. **大规模股票池**（502 只）：信号被噪声淹没
2. **全球最高效的市场**（标普 500）：EMH 效应最强
3. **原始次日收益作为标签**：26.5% 的事件是噪声
4. **FinBERT 处理短标题**（约 15 个英文单词）：信号太浅
5. **事件级别预测**：没有跨时间聚合来增强信号

**代码没有错，数据也没有错。是这个任务本身在这个设定下不可能做好。**

# Signal Fix — 三管齐下的信号修复

在确认 AUC ≈ 0.50 不是 Bug 之后，我们并没有立刻放弃。文献调研同时也指出了几个可能改善信号的方向。我们设计了三项"信号修复"措施。

## Phase 1a：新闻去重（Cell 4）

### 问题

同一只股票在同一天可能有多条新闻。每条新闻都会生成一个独立的训练样本，但它们对应的标签（第二天涨还是跌）是**完全一样的**。这意味着模型看到了多个"看起来不同但答案相同"的样本，其中噪声远多于信号。

### 解决方案

把同一只股票同一天的所有新闻**合并成一条记录**。合并的方法是"平均池化"（mean-pooling）——把所有新闻的向量表示取平均值。

技术实现上，我们用了**稀疏矩阵**（sparse matrix）来高效完成这个平均操作：

```python
# phase_c_model_training.ipynb, Cell 4
# 构建稀疏平均矩阵：W[g, i] = 1/|group_g| 如果事件 i 属于组 g
W = sparse.csr_matrix(
    (np.float32(1.0) / counts[gid].astype(np.float32),
     (gid, np.arange(orig_count))),
    shape=(n_groups, orig_count), dtype=np.float32
)

# 矩阵乘法一步完成所有组的平均
emb = W @ np.asarray(emb[:orig_count], dtype=np.float32)
sent = W @ np.asarray(sent[:orig_count], dtype=np.float32)
```

稀疏矩阵（sparse matrix）是一种专门处理"大部分格子是空的"矩阵的数据结构——在我们的场景中，170 万条新闻要分成 43 万组，如果用普通矩阵需要 7000 亿个格子（内存直接爆掉），但其中只有 170 万个格子非零。稀疏矩阵只存储这 170 万个非零值。

`W @ emb` 这一行——`@` 是矩阵乘法运算符。矩阵乘法的效果就是：对每个组，把组内所有新闻的 768 维向量按权重加起来——由于权重都是 `1/n`，结果就是平均值。

### 结果

```
1,698,182 条新闻事件 → 437,194 条股票-日记录（压缩比 3.88:1）
```

## Phase 1b：市场调整标签（Cell 5）

### 问题

原始标签"这只股票明天涨了吗？"被市场整体方向（专业术语叫 **beta**）主导。假设某天整个市场上涨了 2%，那么绝大多数股票都会跟着涨——标签几乎全是 1，但这不是因为新闻有预测力。

### 解决方案

**什么是市场收益率（market return）？** 所有股票收益率的平均值。**什么是超额收益（excess return）？** 个股收益减去市场收益。新标签：`excess_return > 0`，即"这只股票是否**跑赢了**市场？"

```python
# phase_c_model_training.ipynb, Cell 5
market_ret = rets_all.mean(axis=1)  # 等权重市场收益率
events['excess_return_next'] = events['return_next'] - events['market_return_next']
events['label'] = (events['excess_return_next'] > 0).astype(int)
```

| 指标 | 原始标签 | 市场调整后 | 变化 |
|------|---------|-----------|------|
| 正例比率 | 0.5164 | 0.4925 | 更接近平衡 |
| 噪声区（|收益率| < 0.5%） | 27.6% | 23.0% | 减少 4.6pp |

## Phase 1c：动量特征（Cell 6）

### 什么是动量

动量（Momentum）是金融学中一个被广泛验证的现象：**近期表现好的股票倾向于继续表现好**。

### 9 个特征

3 个时间窗口（5/10/21 交易日） × 3 种统计量（平均日收益率、波动率、累积收益率）。

```python
# phase_c_model_training.ipynb, Cell 6
for w in [5, 10, 21]:
    feat[f'ret_mean_{w}d'] = returns.rolling(w).mean().shift(1)  # shift(1) = T-1!
    feat[f'ret_std_{w}d'] = returns.rolling(w).std().shift(1)
    feat[f'momentum_{w}d'] = prices.pct_change(w).shift(1)
```

注意每一行最后都有 `.shift(1)`——这是**最关键的一步**，把所有数据向后推移一天，确保只使用到**前一天收盘**（T-1）为止的数据，严格杜绝**前瞻偏差**（look-ahead bias，训练时"偷看"了未来数据，是量化金融研究中最致命的错误之一）。

# v3 换赛道 — 从二分类到排名预测

## 为什么要彻底换方向？

经过对 MASTER（AAAI'24）、FinMamba（2025）、MDGNN（AAAI'24）等 10+ 篇最新论文的系统调研，我们发现自己在三个维度上都与学术主流做法相悖。v3 做了三个根本性改变。

### Change 1: 从二分类（涨/跌）到排名预测（谁比谁好）

**旧方法**：对每一条新闻，预测"明天会涨还是跌？"——二分类问题，用 AUC 衡量。

**新方法**：每天对全部 502 只股票，预测"谁会比谁表现更好？"——排名问题。

#### 什么是 IC（Information Coefficient，信息系数）？

IC 是新方法的核心评估指标：**每一天，计算模型预测排名与实际收益率排名之间的 Spearman 秩相关系数，然后对所有天取平均**。

Spearman 秩相关系数比较的是两组**排名**的吻合程度：+1 = 完全一致，0 = 完全随机，-1 = 完全相反。IC 一般在 0.01-0.10 之间就算有用，我们的 Go 阈值设为 IC > 0.03。

#### 什么是 ICIR？

ICIR = IC均值 / IC标准差。衡量 IC 的**稳定性**。ICIR > 0.3 通常被认为可实用。

#### 为什么排名比涨跌更好预测？

在高效市场中，预测"明天是涨还是跌"极其困难——大盘方向主要是随机的。但**相对排名**要容易得多：即使不知道明天市场涨跌，仍然可以判断"科技股可能比公用事业股表现好"。

### Change 2: 从事件驱动到日历驱动

**旧方法**：每有一条新闻就做一次预测。有些股票一天 10 条新闻，有些连续几周没有。

**新方法**：每天为每只股票生成一条预测——502 × ~1250 天 ≈ 62.75 万个样本。没有新闻的天数用零向量 + `has_news=0` 标记。

### Change 3: 从单一次日到多期限预测

**旧方法**：只预测次日收益。

**新方法**：同时预测 6 个期限：1天、5天、10天、21天、42天、63天。

**为什么？** 没有任何论文系统对比过 GNN 在不同预测期限上的表现——这成为我们论文的**核心贡献之一**。

## 新的标签定义

三步处理：

1. **前向收益率**：`close[T+h] / close[T] - 1`
2. **超额收益率**：减去等权市场平均
3. **Z-score 标准化**：每天减均值除标准差 → 均值=0，标准差=1。确保不同波动程度的交易日具有同等权重。

# N3 — 模型对决：谁能预测股票排名？

> 对应 `v3_ranking_pipeline.ipynb` Cell 10-14

## 评估指标（Cell 10）

- **IC**：每天计算预测排名与实际排名的 Spearman 相关，取平均
- **多空组合**：买入排名前 30、卖空排名后 30
- **Sharpe Ratio** = 年化收益 / 年化波动率 × √(252/horizon)
- **交易成本**：15 bps 往返

### ⚠️ 已知缺陷

**Sharpe 是基于 gross return（交易成本前）计算的。** 对 1d horizon，年交易成本 = 252 × 0.15% × 2 = **75.6%**。一个净亏 41% 的策略报告 Sharpe = 2.47——这是误导性的。

## N3a: 非 GNN 基准线（Cell 11）

- **B1: Ridge 回归**（岭回归，带 L2 正则化的线性模型）— 仅价格 9 维
- **B2: Ridge** — 全部 781 维
- **B3: XGBoost**（梯度提升决策树，每棵新树纠正前面的错误）
- **B4: LightGBM**（类似 XGBoost，用直方图分裂加速）

```python
# v3_ranking_pipeline.ipynb, Cell 11
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train_raw)  # 从训练数据学习均值/标准差
X_test = scaler.transform(X_test_raw)         # 用训练数据的参数转换测试数据
```

`fit_transform` 只在训练集调用——测试集只能用训练集学到的参数，否则产生数据泄露。

## N3b: GNN 模型架构（Cell 12）

### HGT（异构图 Transformer）

- **Transformer**：通过"注意力"机制自动学习关注哪些输入
- **HGTConv**：能区分不同边类型的注意力
- 2 层、4 头注意力、hidden=64
- 残差连接（输入直接加到输出）+ LayerNorm（层归一化，稳定训练）

### RankingGNN（同构 GNN）

支持 GraphSAGE（均等聚合邻居）和 GAT（学习不同邻居的注意力权重）：

```python
# v3_ranking_pipeline.ipynb, Cell 12
class RankingGNN(nn.Module):
    def __init__(self, in_channels, hidden, conv_type='gat'):
        if conv_type == 'sage':
            self.conv1 = SAGEConv(in_channels, hidden)
        elif conv_type == 'gat':
            self.conv1 = GATConv(in_channels, hidden // 4, heads=4)
        self.rank_head = nn.Sequential(
            nn.Linear(hidden, hidden), nn.ReLU(), nn.Dropout(0.3),
            nn.Linear(hidden, 1)
        )
```

`rank_head` 将 64 维向量转换成一个标量——即排名预测分数。

## N3c: 训练过程（Cell 13）

- **梯度累积**（Gradient Accumulation）：累积 32 天梯度后更新一次参数，模拟更大批次
- **MSE 损失**：预测 z-score 连续值，不是二分类
- **ReduceLROnPlateau**：验证损失停滞时自动降低学习率
- **早停 patience=15**
- **梯度裁剪 max_norm=1.0**：防止梯度爆炸

## N3d: 消融实验结果（Cell 14）

| 实验 | 模型 | 边类型 | 测试 IC |
|------|------|--------|--------|
| A1 | HGT | 仅相关性 | 0.01023 |
| A2 | HGT | 相关性 + 行业 | 0.01177 |
| A3 | HGT | 全部 4 种 | 0.00432（**最差！**） |
| A4 | SAGE | 相关性 + 行业 | 0.01571 |
| A5 | GAT | 相关性 + 行业 | **0.02054（最佳！）** |

**三个关键发现**：

1. **新闻边有害**：A3（全部 4 种边）最差。噪声连接稀释了有用信号。
2. **简单模型更好**：GAT > SAGE > HGT。弱信号下复杂模型反而过拟合。
3. **图结构有真实价值**：最佳 GNN IC = 0.02054 vs 最佳基准 IC = 0.00828，**2.5× 提升**。

**4. 1d Sharpe 虚假繁荣**：毛年化 +34.5%，扣费 -75.6%，净 = **-41%**。

**这个倒U型规律从未在 GNN 文献中被系统展示过——是原创科学发现。** GAT 21d IC=0.044 超过 0.03 门槛。

# N5 — 选择性预测（SelectiveNet）

> 对应 `v3_ranking_pipeline.ipynb` Cell 16-18

## 核心思想

不是所有预测都同样可靠。**SelectiveNet**（ICML 2019）学习**何时预测、何时弃权**。三个头：预测头、选择头、辅助头。

**此前从未有人将 SelectiveNet 与 GNN 结合应用于股票预测。**

## 架构（Cell 16）

- **共享 GAT 骨干**：和 N3 相同
- **排名头**：输出排名分数
- **选择头**：`[embedding; market_context(4维)]` → sigmoid → 选择概率

4 个市场环境特征（全部 T-1）：
1. 21 天已实现波动率（**VIX 替代指标**——未获取真实 VIX 数据）
2. 63 天回撤深度
3. 30 天横截面波动率均值
4. 5 天市场宽度

- **辅助头**：提供正则化

## 选择性损失函数

```python
# v3_ranking_pipeline.ipynb, Cell 16
selective_risk = (selection * (pred - target)**2).sum() / selection.sum()
coverage_penalty = lambda_ * F.relu(target_cov - selection.mean())**2
loss = selective_risk + coverage_penalty + alpha * aux_mse
```

- **selective_risk**：加权 MSE，选择概率高的股票贡献更多损失
- **coverage_penalty**：鼓励选择约 20% 的股票
- **aux_mse**：辅助头正则化

## 两阶段训练（Cell 17）

**Stage 1**（100 轮）：训练骨干 + 排名 + 辅助，普通 MSE（无选择）。

**Stage 2**（50 轮）：冻结骨干和排名头，只训练选择头。

分两阶段的原因：防止骨干"作弊"——不提高预测准确性，而是降低难预测股票的选择概率。

## 结果：SelectiveNet 失败

| 方法 | IC | Sharpe | 年化收益 |
|------|-----|--------|---------|
| 完整预测（100%） | **0.056** | 1.328 | 16.48% |
| 阈值筛选（20%） | 0.031 | 0.724 | 5.85% |
| SelectiveNet（20%） | **-0.024** | -0.536 | **-9.00%** |

**完整预测反而最好**——IC=0.056（辅助头的正则化效果）。

**SelectiveNet 学到了反向选择**——系统性选中 GNN 预测最不准的股票。

### 失败原因

1. **覆盖率未收敛**：目标 20%，实际 31%。lambda=32 不足。
2. **选择分数无区分度**：集中在 0.8-1.0，无法区分可靠/不可靠预测。
3. **排名信号太弱**：IC=0.04 下，"容易"和"难"预测的差异不够明显。
4. **市场环境特征有限**：4 维且是替代指标，信息不足。
5. **两阶段局限**：选择头无法影响 embedding 生成。

# 训练稳定性问题

GAT IC 跨两次运行从 0.006 变到 0.021——**变异系数 CV = 105%！** SAGE 的 CV 只有 2%。

不稳定来源：
- 随机初始化
- 训练日随机打乱
- 梯度累积边界
- 早停敏感性

**单次实验结果可能不代表模型真实能力。**
